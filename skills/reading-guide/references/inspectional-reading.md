# 检视阅读操作手册

> Step 2 每次必用。这是最频繁查阅的 reference。

---

## 检视阅读的两种子技法

### 一、系统略读（7 步）

目标：在 15-30 分钟内，对一本书的架构和价值做出判断。

```
1. 看书名页 + 序言
   → 快速看过。注意副标题、作者的宗旨或特殊视角。
   → 此时你应该已经能将这本书归类到某个特定类型。

2. 研究目录页
   → 对基本架构做概括性理解。就像出发前先看地图。
   → 大多数人不看目录，但目录是作者花了心思写的。

3. 检阅索引（如果有）
   → 快速评估这本书涵盖了哪些议题范围。
   → 如果发现某个重要词汇，翻到引用该词的几页看看。

4. 读出版者介绍（书衣/腰封）
   → 很多书的宣传文案是作者亲自写的摘要。
   → 如果只是瞎吹牛，你也能轻易看出来。

5. 挑几个看来跟主题息息相关的篇章来看
   → 如果这些篇章在开头或结尾有摘要说明，仔细读。

6. 东翻西翻，跳读几段
   → 不要太多。留意主要论点的信号，追踪主题的基本脉动。
   → 尤其不要忽略最后两三页——作者很少能抗拒在那里总结自己最新最重要的观点。

7. 回答三个问题
   → 这是哪一类书？
   → 这本书在谈什么？
   → 作者是借着怎样的整体架构来发展他的观点？
```

### 二、粗浅阅读

**规则**：头一次面对一本难书，从头到尾读完一遍，碰到不懂的地方**不要停下来查询或思索**。

**为什么要这样做**：
- 只注意你能理解的部分，不要为没法立刻了解的东西停顿
- 继续读下去，略过不懂的部分，你很快就会读到看得懂的地方
- 读完一遍之后的理解（哪怕只有 50%），能帮你在重读时增进理解
- 如果一碰上困难就停住，最后可能对这本书真的一无所知

**适用场景**：
- 第一次读一本难度很高的书
- 阅读文学经典（如莎士比亚戏剧）——不要在第一次读时逐字查注释
- 阅读理论经典（如《国富论》）——不要坚持每一页都懂了才往下

## 阅读速度

> "读得太快或太慢，都一无所获。" ——帕斯卡尔

- 检视阅读时，只读书中的一小部分，且用不同的速度来读
- 不是所有书都值得用同样的速度
- 一本只需要快速阅读的书，用很慢的速度读 = 浪费时间
- 好的速读课程应该教你**不同的阅读速度**，而非一味求快

## 检视阅读能回答 & 不能回答的问题

| 能回答 | 不能回答 |
|--------|---------|
| 这是哪一类书？ | 作者的核心主旨是什么？（需要阶段二） |
| 这本书整体在谈什么？ | 作者的论证是否成立？（需要阶段三） |
| 它的架构如何？ | 这本书说得有道理吗？（需要阶段三） |
| 它值得我花时间做分析阅读吗？ | 这本书跟我有什么关系？（需要阶段三） |
| 作者想要解决什么问题？ | 关键术语的确切含义？（需要阶段二） |

**诚实边界**：检视阅读只能回答 Q1 的粗粒度版本。不要跨越这个边界。如果你不得不做推测，明确标注「推测」和不确定性。

## Inspectional Report 输出约束

- **字数**：≤400 字
- **内容**：分类 → 一句话概括 → 结构骨架（≤5 节点）→ 核心问题（2-4 个）→ 关键概念预览 → 阅读策略 → 阅读时带的问题（3-5 个）
- **格式**：使用 `templates/inspectional-report.md`
- **如果只有书名无原文**：强制标注「⚠️ 二手判断，精度有限」
- **如果是虚构类**：切换到 `templates/fiction.md` 预读部分

---

## 检视阅读不适用的情况

- 用户**已经读完了**这本书 → 直接进入分析阅读（Step 4），跳过检视
- 用户提供了**自己的大量划线** → 已经有分析阅读的数据基础，检视仍然有价值（帮你回顾整体结构），但不是必需的
