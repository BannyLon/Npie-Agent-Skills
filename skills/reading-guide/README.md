<p align="center"><strong>不是替你读完一本书，而是教你学会自己读。</strong></p>

## 这是什么

**reading-guide** 是一个 Claude Code Skill，基于莫提默·J·艾德勒 & 查尔斯·范多伦的经典著作《如何阅读一本书》(_How to Read a Book_)，将书中的**四层次阅读法**转化为可执行的 AI 阅读教练。

它会：

- 在你翻开书之前，帮你建立认知框架——这本书在谈什么？结构如何？该怎么读？
- 在你读完之后，带你走完分析阅读的三个阶段——结构 → 诠释 → 评论
- 最终让你**自己**回答那个最重要的问题：**这本书跟我有什么关系？**

它不会：

- 替你读完一本书，然后给你一个漂亮的摘要
- 告诉你"这本书好不好"然后让你把它当作自己的判断
- 跳过真正的阅读

> 如果用户拿着报告就可以说「我懂了，不用读了」——那这个 skill 就失败了。它的成功是：你读完报告后更想读这本书，而且知道该怎么读。

---

## 理论基础

这个 skill 是对艾德勒《如何阅读一本书》的完整工程化实现。

| 框架 | skill 中的对应 |
|------|---------------|
| **四个阅读层次**（基础→检视→分析→主题） | 五步工作流，检视默认必做，分析三阶段递进，主题阅读支持多本 |
| **四个基本问题** | 每份报告的每一部分都锚定一个问题 |
| **分析阅读 15 条规则** | 三阶段严格递进（结构→诠释→评论），不可跳 |
| **核心纪律：理解优先于评判** | 在说出「我了解了」之前，不准说「我同意/不同意」 |
| **6 类书不同读法** | 实用型 / 想象文学 / 历史 / 科学 / 哲学 / 社会科学，各有独立路径 |
| **书的金字塔** | 帮你判断一本书在金字塔的哪一层——99.9% 只需扫读，不到 100 本值得一生反复读 |

## 快速开始

### 前置要求

- [Claude Code](https://claude.ai/code)（CLI 或 IDE 扩展）
- （可选）微信读书 API Key——如果你用微信读书，skill 可以拉取你的划线和笔记做个性化分析。不用也完全可以，skill 会通过 Web 搜索获取公开资料
- （可选）[Obsidian](https://obsidian.md)——模板用 Mermaid 渲染思维导图，Obsidian 原生支持

### 安装

```bash
# 克隆到 Claude Code 的 skills 目录
git clone https://github.com/{your-org}/reading-guide.git ~/.claude/skills/reading-guide

# 或者在 Claude Code 中直接添加 skill
claude skills add reading-guide --from github:{your-org}/reading-guide
```

### 设置（可选）

如果你使用微信读书，配置环境变量：

```bash
export WEREAD_API_KEY="your_api_key_here"
```

不设置也完全没问题——skill 会自动降级到 Web 搜索。

## 使用示例

### 场景 1：准备读一本书

```
用户：我想读《思考，快与慢》，怎么读？

skill：
  → 搜索获取目录和热门划线
  → 判断类型：非虚构 → 理论型 → 科学（认知心理学）
  → 产出检视报告：

  ## 《思考，快与慢》检视阅读报告

  ### 作者要解决的核心问题
  1. 人类决策为什么经常出错？（理论性）
  2. 大脑的两套思考系统——系统1和系统2——如何协作和冲突？（理论性）
  3. 了解了这些偏差之后，我们如何做出更好的决策？（实用性）

  ### 检视判断
  | 架构质量 | 清晰——五部分递进，每部分有明确的主题 |
  | 论证可信度 | 高——基于诺贝尔奖得主40年实证研究 |
  | 难度 | ⭐⭐⭐ 概念不难，但信息密度大 |
  | 数据来源 | 🟡 Web 搜索 |

  ### 初步印象
  > 精读。但要跳读——前两部分（系统1和系统2的核心理论）需要精读，
  > 第三四部分（各类偏差案例）可以跳读，第五部分（两个自我的区分）回来精读。

  ⭐ 从外部检视来看，我的初步印象是：精读
  但最终判断权在你。

  你要：A 按初步印象走 | B 我就是想精读 | C 只精读某几章 | D 横评
```

### 场景 2：读完了，想消化

```
用户：刚读完《也许你该找个人聊聊》，帮我整理一下

skill：
  → 拉取你的 2 条划线 + 热门划线 + 目录
  → 启动分析阅读三阶段

  阶段一·结构：
    分类：非虚构 → 实用型（叙事）
    一句话：一位治疗师通过自己接受治疗的故事 + 四个患者的故事，
            展示"改变如何在真实的关系中发生"
    结构：四部分递进（崩塌→深入→对峙→整合）

  阶段二·诠释：
    关键术语：痛苦/受苦、改变、承认、负责、完美主义
    核心主旨：改变不是靠道理和技巧，而是在关系中承认真相
    论证：叙事归纳式——5个案例交叉验证同一组规律

  阶段三·评论：
    知识不足：缺乏神经科学和CBT视角
    知识错误：不适用（叙事书不做科学论断）
    不合逻辑：选择偏差——只展示了成功的案例
    分析不完整：没有讨论"没有治疗师的人怎么办"

  🎯 这本书跟你有什么关系？
    1. 这些故事里有没有你的影子？在哪个角色身上看到了自己？
    2. 你目前在逃避什么改变？
    3. 为此你需要先放手的是什么？
    （答案必须是你的，不是我的）
```

### 场景 3：几本书里选一本

```
用户：《原子习惯》《掌控习惯》《微习惯》哪本值得读？

skill：
  → 三本各自检视 → 对比表：

  | 维度 | 原子习惯 | 掌控习惯 | 微习惯 |
  |------|---------|---------|--------|
  | 类型 | 实用型 | 实用型 | 实用型 |
  | 核心论点 | 身份驱动习惯 | 环境设计 | 小到不可能失败 |
  | 难度 | ⭐ | ⭐ | ⭐ |
  | 适合谁 | 想系统化改习惯 | 想优化环境 | 坚持不下去的人 |

  ⭐ 如果只能读一本：《原子习惯》——体系最完整，且"身份驱动"
    这个视角是另外两本没有的。
```

### 场景 4：小说

```
用户：怎么读《百年孤独》？

skill：
  → 识别为虚构类 → ⛔ 停下来：
  "这是想像文学。按艾德勒的说法，不要去找共识/主旨/论述——这
   是经验的传递，不是知识的传递。全身心投入体验。要我切换吗？"

用户：要

skill：
  → 情节大意 → 叙事结构（环形时间）→ 进入这个世界 → 阅读策略
    （快读，不要被重复的名字吓退，不要做人物关系图——让故事
    在你身上发生）
```

## 工作流

```
第一步：断类型，分路径
    来源瀑布式获取（微信读书 → Web 搜索）→ 虚构/非虚构分叉 → 细分类型
    ↓
第二步：检视阅读（默认必做）
    产出 Inspectional Report：核心问题 + 检视判断 + 初步印象
    ↓
第三步：用户选择（必须停止！）
    四个选项：
    A 按初步印象走 | B 我就是想精读 | C 只精读某几章 | D 主题阅读/横评
    ↓ 用户选了才继续
第四步：分析阅读 → 三阶段严格递进
    阶段一·结构 → ⏸️ → 阶段二·诠释 → ⏸️ → 阶段三·评论
    每个阶段末尾引导用户自己做出判断
    ↓
第五步：结构化交付
    Markdown → outputs/ → 提示 HTML 视觉版
```

**三个用户确认闸门**确保你不会被自动推进。

## 输出格式

每份报告包含四件套：📇 快照卡片 · 🧠 思维导图 · 📋 行动清单 · 📚 延伸阅读。

默认交付 Markdown（存于 `outputs/`），可选转 HTML（5 种风格：学术严谨 / 数字杂志 / 极简纯净 / 暗夜影院 / Obsidian 原生）。

## 与同类工具的区别

| 工具 | 做什么 | reading-guide 的不同 |
|------|--------|---------------------|
| AI 总结一本书 | 帮你"读完" | 教你**学会自己读** |
| 书评 / 读书笔记 | 告诉你别人怎么想 | 帮你**形成自己的判断** |
| 微信读书划线导出 | 导出你画的金句 | 帮你**把金句变成结构化的理解** |
| 阅读打卡/计时 | 跟踪读了多久 | 关注**读了多少不是重点，理解了多少才是** |

## 常见问题

### 我没有微信读书，能用吗？

完全可以。给一个书名就行。skill 会通过 Web 搜索获取公开资料来完成检视阅读。微信读书只是一个可选的数据增强——如果你在用，skill 可以拉取你的划线和笔记做个性化分析。

### 只能用 Claude Code 吗？

目前 skill 是为 Claude Code 设计的。但它是一个纯 Markdown 格式的 skill，理论上可以迁移到任何支持 skill 机制的 AI 平台。

### 检视报告 ≤400 字够用吗？

这个限制是有意为之。检视阅读的目的是**快速判断**，不是深读。400 字足以覆盖一本书的分类、核心问题、结构和初步印象。如果你需要深度分析，那就是分析阅读做的事了。

### 分析阅读三个阶段一定要全部走完吗？

不用。你可以在任何时候叫停。三个阶段递进但独立——走完阶段一就比大多数人读得更深了。只是别忘了：没做完阶段一不准进阶段二，没做完阶段二不准进阶段三。

### 这个 skill 适合所有书吗？

不是。按艾德勒的书之金字塔——99.9% 的书只需要检视阅读。只有那些架构清晰、值得咀嚼与消化的书才需要分析阅读。skill 的检视步骤就是帮你做这个判断的。
