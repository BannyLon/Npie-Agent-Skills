# wechat-automator v1.1.0

> 一站式微信公众号内容资产化引擎 — 把短期流量（视频/笔记/碎片资料）转化为长期内容复利。

`wechat-automator` 是一个 Claude Code Skill，它将**内容深度优化**、**精排版渲染**、**一键推送草稿箱**整合为一条自动化流水线。你说一句「发公众号」，它从 Markdown 到草稿箱全部搞定。

---

## 它能做什么

```
你给一个文件 + 说一句「发公众号」
         ↓
  ① 内容脱水：去掉口语填充，提炼核心观点
  ② 结构重组：重构钩子开头、优化段落节奏、强化 CTA
  ③ 视觉增强：列表卡片化、金句模块提取、截图位标记
  ④ 排版渲染：零 div + 全 inline style（微信安全 HTML）
  ⑤ 一键上传：自动生成封面 → 推送公众号草稿箱
         ↓
  打开后台 → 预览 → 群发 ✅
```

**核心差异化能力**：不仅仅是排版工具，它会主动分析内容结构，自动识别哪些地方该加卡片、金句、截图位，生成「图文并茂」的杂志级排版，而不是一面文字墙。

## 快速开始

### 安装

1. 将 `wechat-automator/` 目录放入你的 Claude Code skills 目录：

```bash
cp -r wechat-automator/ ~/.claude/skills/wechat-automator/
```

2. （可选）设置环境变量以避免每次输入凭据：

```bash
export WEIXIN_APPID="你的公众号 AppID"
export WEIXIN_APPSECRET="你的公众号 AppSecret"
```

凭据获取路径：微信公众号后台 → 开发 → 基本配置。

### 触发方式

说人话就行，不需要精确指令：

| 你说的话 | 效果 |
|----------|------|
| 「帮我把这篇排版发公众号」 | 完整链路：优化 → 排版 → 上传 |
| 「这个脚本转成推文」 | 完整链路 |
| 「排版看看效果，先不发」 | 仅排版预览，输出 HTML 文件 |
| 「严格按原文排版发草稿箱」 | 跳过内容修改，仅排版上传 |
| 「仅排版，不需优化内容」 | 跳过内容修改，仅排版上传 |

> ⚠️ **重要**：触发后 Claude 会自动调用 `wechat-automator` skill 并严格按其内部五阶段流水线执行。整个过程是自动化的，你不需要手动运行任何脚本。

### 豁免内容修改

如果你不想让 AI 改动你的原文，使用以下任一关键词即可跳过内容优化阶段（排版和视觉增强仍然执行）：

- `一字不改` / `原封不动` / `严格按原文`
- `不要修改内容` / `不做任何修改` / `不要改动`
- `仅排版` / `只排版` / `纯排版`
- `不增不减` / `保留原文` / `直接排版`

## 工作流程（五阶段）

### ① 内容脱水 — 可豁免

剥离镜头语言、口头禅、重复论述，将散落叙述收敛为核心观点句。目标：信息密度提升 30-50%，同时保留作者的标志性表达风格。

### ② 结构重组 — 可豁免

将线性内容重构为适合图文阅读的层级化结构：
- **钩子开头**：2-3 句制造信息落差
- **正文骨架**：从教程/观点/案例/清单四种结构中选择
- **结尾 CTA**：给读者一个具体的下一步

### ③ 视觉增强 — 始终执行

自动识别配图触发点并插入对应元素：

| 触发场景 | 插入元素 |
|----------|----------|
| 关键概念定义 | 概念卡片 |
| 步骤流程 | 流程卡片（每步一张） |
| 数据/对比 | 响应式表格 |
| 重点结论 | 金句模块 |
| 工具推荐 | 资源卡片 |
| 界面操作 | 截图占位符 |

### ④ 排版渲染 — 始终执行

基于微信生产环境实测得出的硬约束：

| 规则 | 原因 |
|------|------|
| 零 `<style>` 标签 | 微信完全删除 |
| 零 `class` 属性 | 微信完全剥离 |
| 零 `<div>` 元素 | 微信转为裸 `<p>`，样式全部丢失 |
| 全 `style=""` 内联 | 唯一可靠的样式方式 |
| 纯色背景 | `linear-gradient` 会被微信删除 |
| 语义元素替代 | `section` / `blockquote` / `table` / `p` |

### ⑤ 一键上传 — 按需执行

凭据获取（优先级）：环境变量 → 会话缓存 → 主动询问 → 自动生成品牌色封面 PNG → 上传素材库 → 创建草稿 → 返回 media_id。

## 常见问题

### 标题/摘要显示不全或上传报错？

微信 API 对标题和摘要按**字节数**（不是字符数）校验。中文字符在 UTF-8 下占 3 字节，标题实际上限约 35 字节（≈12 个纯中文）。如果标题超限，skill 自动用短标题上传，你在公众号后台可手动改回完整版——后台编辑器没有字节限制。

### 草稿箱内容乱码？

确保使用最新版 skill。旧版存在 JSON 序列化问题（`requests` 的 `json=` 参数默认将中文转为 `\uXXXX` 转义序列），会导致草稿箱中文全部乱码。最新版已修复此问题。

### 可以只排版不上传吗？

可以。说「排版看看效果，先不发」即可。skill 输出排版后的 HTML 文件供本地预览。

## 安全与隐私

- API 凭据仅从环境变量或会话内存获取，**不写入任何磁盘文件**
- 会话结束后凭据自动丢弃，下次需重新提供（或通过环境变量持久化）
- 生成的 HTML 和草稿内容保存在本地，不经过第三方服务中转
- 上传完成后自动清理临时文件（JSON payload、自动生成的封面 PNG）
- 凭据在日志和输出中不出现明文
